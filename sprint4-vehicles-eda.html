Used Cars Explorer:
Interactive EDA Dashboard

By Viet Nguyen  ·  Completed 2024  ·  6 min read

← Back to Projects
01

Overview

This project is a fully deployed, interactive EDA dashboard built with Streamlit and hosted on Render. It lets anyone explore a U.S. used-car listings dataset without writing a single line of code — users toggle charts on and off via a sidebar and interact with Plotly visualizations directly in the browser.

The underlying dataset contains thousands of vehicle listings with fields for price, odometer reading, model year, and condition. The four charts together tell a coherent story about how these variables relate — from price distributions to the effect of mileage and age on resale value.

The live app is deployed at tripleten-sprint4-ww34.onrender.com — no setup required.
02

The Four Charts

Chart 1

Distribution of Vehicle Prices

A histogram with 50 bins showing the full price range across listings. The distribution is heavily right-skewed — most listings cluster below $20,000, with a long tail of luxury and collector vehicles. This view immediately surfaces the typical price range a buyer should expect.

Chart 2

Distribution of Vehicle Model Years

A histogram of model years showing which vintages dominate the used-car market. Recent model years (2010s and newer) account for the bulk of listings, with older vehicles becoming increasingly rare — useful context for understanding supply and demand by era.

Chart 3

Price vs. Odometer Reading

A scatterplot of price against odometer reading. The expected inverse relationship holds: higher mileage correlates with lower price, though with significant spread — brand, condition, and model year all add variance. The semi-transparent points (opacity=0.6) reveal density in heavily clustered regions.

Chart 4

Price vs. Model Year by Condition

A color-coded scatterplot adding vehicle condition as a third dimension. Newer, like-new condition vehicles sit in the top-right corner (high price, recent year), while older vehicles in fair or salvage condition cluster bottom-left. This chart makes it easy to spot value outliers — well-priced newer vehicles or overpriced older ones.

03

Implementation

The app is a single app.py file. Streamlit's checkbox widgets drive which charts render — each chart is wrapped in a conditional block, so only the selected visualizations are computed and displayed. Data is loaded once and cached with @st.cache_data to avoid re-reading the CSV on every interaction. 

import streamlit as st
import pandas as pd
import plotly.express as px

@st.cache_data
def load_data():
    return pd.read_csv("vehicles_us.csv")

df = load_data()

show_price_hist = st.sidebar.checkbox("Histogram of Vehicle Prices")

if show_price_hist:
    fig = px.histogram(df, x='price', nbins=50,
                        title='Distribution of Vehicle Prices',
                        labels={'price': 'Price ($)'})
    st.plotly_chart(fig, use_container_width=True)

Deployment to Render required only a requirements.txt listing the dependencies — Render detects the Streamlit app automatically and spins up a web service. No Dockerfile or server configuration needed.

app.py Streamlit dashboard — all UI and chart logic

04

What I Learned

  • Streamlit's reactive model — the entire script re-runs on every widget interaction; @st.cache_data is essential for keeping data loading fast without redundant I/O.
  • Plotly Express for EDA — Plotly charts are interactive out of the box: hover tooltips, zoom, pan, and legend toggles with zero extra code.
  • Deploying to Render — pushing a Streamlit app to production is as simple as connecting a GitHub repo and specifying the start command. The CI/CD pipeline redeploys automatically on every push.
  • Data storytelling — presenting four charts in a logical sequence (distribution → distribution → relationship → relationship + dimension) guides the user toward a coherent understanding of the dataset rather than a random set of graphs.

Key Takeaway

Streamlit collapses the gap between a Jupyter notebook and a deployed web app. The same Pandas and Plotly code that runs in a notebook can be wrapped in a shareable URL with minimal overhead — making it easy to deliver EDA findings to a non-technical audience.
